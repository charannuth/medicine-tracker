# Contributing to Medicine Tracker

Thank you for your interest in contributing. This project is in early development; guidelines will evolve as the codebase grows.

## Getting started

1. Fork the repository and clone your fork.
2. Create a branch from `main` for your work (`feature/…`, `fix/…`, or `docs/…`).
3. Once the app is initialized, follow the setup steps in the root README.
4. Open a pull request against `main` with a clear description of changes.

## Pull requests

- Keep changes focused; prefer smaller PRs over large mixed ones.
- Update documentation when behavior or setup changes.
- Add or update tests when the test harness exists.
- Ensure CI passes before requesting review.

## Commit messages

Use clear, imperative subjects:

- `Add medication schedule screen`
- `Fix duplicate dose log for same day`
- `Update README setup instructions`

## Code style

Follow existing patterns in the repo. When the mobile app lands:

- TypeScript for application code
- Consistent formatting (Prettier / ESLint will be configured with the project)

## Reporting issues

Use the GitHub issue templates for bugs and feature requests. Include steps to reproduce, expected vs actual behavior, and environment details (OS, app version).

## Medical & safety disclaimer

This app is not a substitute for professional medical advice. Contributors must not present the app as a diagnostic or treatment tool. User-facing copy should encourage users to consult healthcare providers for medical decisions.

## Questions

Open a GitHub Discussion or issue if something is unclear. For security issues, see [SECURITY.md](SECURITY.md) — do not file public issues for vulnerabilities.
